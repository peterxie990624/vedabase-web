
#### 关于主题配色与图片预加载的开发指南：
- **主题配色设计**：不必完全复制现有颜色，应该自主设计配色方案，确保与应用整体风格相匹配，同时为浅色和深色主题都提供相应的配色方案
- **浅色主题配色**：通常采用蓝色系（如 #4a7fa5、#1a3a5c、#e8f2f9 等），与应用的深蓝色主题相协调
- **深色主题配色**：通常采用金色系（如 #c8a84b、#e8d5a3、#0f1923 等），与应用的夜间深色主题相协调
- **图片预加载**：使用 JavaScript Image 对象进行预加载，显示加载动画直到图片加载完成，同时处理加载失败的情况
- **加载动画**：采用旋转的圆形加载指示器，使用 CSS animation 实现平滑的旋转效果
- **UI 命名约定**：不要在界面中显示开发时的临时名称（如 'xxx'），应该使用正式的功能名称
- **可复用性考虑**：提取通用的加载逻辑、配色方案、动画效果等，为后续类似功能的实现提供参考

#### 关于代码推送与部署的开发指南：
- **推送流程**：使用 GitHub Personal Access Token (PAT) 进行身份认证，命令格式为 `git push https://<token>@github.com/<username>/<repo>.git <local_branch>:<remote_branch>`
- **分支管理**：本项目使用 `main` 作为主分支，所有功能完成后推送到 `main` 分支
- **自动部署**：推送到 `main` 分支后，GitHub Actions 会自动触发构建和部署流程（约 1-2 分钟）
- **部署地址**：部署完成后，网站会自动更新到 https://peterxie990624.github.io/vedabase-web/
- **版本检查**：可以通过开发模式工具（DEV 按钮）查看当前部署的版本号（commit hash）和构建时间
- **Token 安全**：Personal Access Token 应该妥善保管，不要在代码中硬编码，仅在推送时使用
