## 修复内容

### 1. 导航返回逻辑（App.tsx 核心重构）

**问题**: 从书签/搜索进入阅读页后，返回时错误地回到章节列表页，而不是书签页/搜索页。

**根本原因**: `handleOpenBookmark` 和 `handleSearchResult` 调用了 `setActiveTab('bookshelf')`，导致 URL 变为 `#/bg/...`，返回时无法知道来源是书签还是搜索。

**解决方案**: 引入 `overlayRoute` 机制：
- 从书签/搜索进入阅读页时，**不切换 activeTab**，而是设置 `overlayRoute`（包含路由和 `returnTab`）
- `renderContent` 优先渲染 `overlayRoute` 中的内容
- `pop()` 检测到 `overlayRoute` 时，关闭 overlay 并切回 `returnTab`（书签页或搜索页）

```typescript
type OverlayRoute = {
  route: Route;
  returnTab: TabType; // 返回时切回哪个 tab
};
const [overlayRoute, setOverlayRoute] = useState<OverlayRoute | null>(null);
```

### 2. Tab 切换记忆

**问题**: 在阅读页切换到其他 tab 再切回书架，应恢复阅读位置，但当前始终回到书架首页。

**解决方案**: 使用 `prevTabRef` 记录上一个 tab：
- 从非书架 tab 切回书架：恢复 `bookshelfRouteStack`（包含上次阅读位置）
- 重复点击书架 tab（`tab === activeTab`）：回到书架首页

```typescript
const prevTabRef = useRef<TabType>(initial.tab);
// 在 handleTabChange 中：
if (tab === 'bookshelf' && prevTab !== 'bookshelf') {
  setRouteStack(bookshelfRouteStack); // 恢复阅读位置
}
```

### 3. 搜索高亮颜色

**问题**: 搜索高亮颜色为红色背景，用户要求改为"悦人的金色"（主柴坦尼亚金色）。

**修改**:
- `index.css`: `.search-highlight { background: #d4a017; color: #1a1a1a; }`
- 暗色模式: `background: #b8860b; color: #fff8e7;`
- `SearchPage.tsx` `highlightText()`: 内联样式也改为金色
- `SectionContent.tsx` `highlightText()`: 已是金色（`background:#ffd700`），保持不变

### 4. 智能截取关键词所在句子

**问题**: 搜索结果预览截取方式简单（居中截取），不够智能。

**解决方案**: `extractPreview()` 改为找到关键词所在句子（以 `.!?\n` 为边界），超长时回退到居中截取。

### 5. 搜索时间戳

记录每次搜索时间到 `localStorage.setItem('vedabase_last_search_time', ...)`，用于 2 小时记忆时限判断。

### 6. 封面图加载优化（BookshelfPage.tsx）

**问题**: 封面图加载过程可见（先显示破图/空白，再显示图片）。

**解决方案**: 
- 使用 `coverLoaded` 状态记录每本书封面是否加载完成
- 图片 `opacity: 0`，加载完成后 `opacity: 1`（0.3s 淡入）
- 加载完成前显示纯色占位背景

### 7. 开发模式激活

**问题**: URL `#dev` 方式无法稳定激活开发模式（hashchange 事件处理复杂）。

**解决方案**: 改为**长按"关于"按钮 3 秒**激活/关闭开发模式：
- 存储在 `localStorage.setItem('vedabase_devmode', 'true')`
- `isDevMode()` 读取 localStorage 或 `import.meta.env.DEV`
- 支持 mousedown/touchstart 长按，mouseup/mouseleave/touchend 取消
