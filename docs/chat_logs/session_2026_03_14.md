# 会话记录 2026-03-14

## 本次完成的工作

### 1. 修复 Hash 路由回退逻辑（App.tsx）

**问题根因**：`hashToState` 函数只返回当前页面，不包含父级页面，导致 `routeStack` 每次 hash 变化后只有 2 层（home + 当前页），再按回退就直接到首页。

**修复方案**：`hashToState` 改为返回完整路由栈（`stack: Route[]`），包含所有父级页面：
- BG: `[home, bg-chapters, bg-sections, bg-read]`
- SB: `[home, sb-cantos, sb-chapters, sb-sections, sb-read]`
- 书签跳转也包含完整父级路由栈

**关键代码变更**：
```ts
// 旧：只返回当前路由
function hashToState(hash): { route: Route; tab: TabType }

// 新：返回完整路由栈
function hashToState(hash): { stack: Route[]; tab: TabType }
```

### 2. 优化 CDN 加载策略（useData.ts）

**旧策略**：先探测 jsDelivr（4s 超时），再决定用哪个源，串行加载。

**新策略**：并行竞速（Race）加载
- 同时向 jsDelivr 和 GitHub Pages 发起请求
- 谁先成功响应就使用谁，另一个被取消（AbortController）
- 去掉了 4s CDN 探测延迟，大幅提升首次加载速度
- 使用 `Promise.any()` 实现竞速

### 3. SB 阅读页添加进度条（SBReadPage.tsx）

- 引入 `LoadingProgress` 组件
- 通过 `onProgress` 回调收集加载事件
- 显示：加载中的文件名、数据来源（jsDelivr/GitHub）、耗时
- 开发模式下显示详细诊断日志

## 技术细节

### Hash 路由层级结构
```
BG:  #/ → #/bg → #/bg/{chId} → #/bg/{chId}/{secIdx}
SB:  #/ → #/sb → #/sb/{cantoId} → #/sb/{cantoId}/c{chId} → #/sb/{cantoId}/c{chId}/{secIdx}
```

注意：SB 的 hash 格式中 `cantoId` 和 `chapterId` 都需要编码，以便刷新后能恢复完整路由栈。旧格式 `#/sb/c{chId}` 作为兼容处理（cantoId 缺失时，父级只有 sb-cantos）。

### 并行加载实现
```ts
const winner = await Promise.any([jsdPromise, ghPromise]);
// 取消另一个请求
if (winner.source === 'jsdelivr') ghCtrl.abort();
else jsdCtrl.abort();
```
