### 第五阶段：修复 Promise.any() 兼容性问题（iPad Safari）

**用户消息：**
现在这个是苹果17的浏览器，你能排查一下原因吗

用户截图显示：Safari 浏览器显示"Safari浏览器打不开该网页，因为已丢失网络连接"

**Manus 操作：**
- 诊断问题：
  1. 验证网站本身可正常访问（HTTP 200）
  2. 发现根本原因：`Promise.any()` 在旧版 iPad Safari 上不支持
  3. `Promise.any()` 是 ES2021 新特性，Safari 15 以下不支持
  4. 导致所有数据加载完全失败，页面显示空白

- 修复方案：
  1. 添加 `Promise.any()` 的兼容性检查
  2. 旧版浏览器自动降级使用 `Promise.race()`

- 代码改动：
  - useData.ts：在 loadCantoData 和其他数据加载函数中添加兼容性检查

- 提交：`e15580c` - fix: 修复Promise.any()兼容性问题

### 第八阶段：诊断中国大陆网络访问问题

**用户消息：**
还有一个人，用vivo手机，不是苹果，但是也没法打开，后来他连接了WiFi，就能正常打开了

用户截图显示：vivo 手机浏览器错误 `net::ERR_CONNECTION_RESET`

**Manus 操作：**
- 诊断问题：
  1. iPhone 5G：Safari 显示"丢失网络连接" → DNS 污染
  2. vivo 移动数据：`ERR_CONNECTION_RESET` → 连接被重置，典型的 DNS 污染
  3. vivo WiFi 后正常 → WiFi 路由器的 DNS 不同

- 根本原因：中国移动/联通/电信的手机数据网络 DNS 对 `*.github.io` 域名进行了污染

- 解决方案建议：
  1. 迁移到 Cloudflare Pages（免费，国内稳定性更好）
  2. 购买自定义域名 + 阿里云/腾讯云 CDN（最稳定）
